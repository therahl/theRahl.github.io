Understand the problem

Before working on a problem, it is important to understand what you are solving. Often, solving a problem that may seem overwhelming is easiest by breaking that problem into smaller, easily solved problems.
- Break the problem into small pieces and know how they work together - Outline the problem with psuedo code

Ask the Right Questions

Recognize underlying assumptions

Get Unstuck

In no particular order: - Work through similar examples. - Break the problem down into simple steps. - If using a gem or library, read the codebase for a better understanding - Ask Google/StackOverflow/mentor - Check log for errors (obvious, but easily overlooked) - Talk to the rubber duck (especially helpful if you have a really smart rubber duck)
